# UCLASS Stuttered Speech Clips (SEP-28k Format): Dataset Guide

## Overview

This dataset provides a processed subset of the **University College London Archive of Stuttered Speech (UCLASS)**, formatted for direct use in machine learning workflows, particularly for stuttering detection. The data has been processed to mirror the structure of the widely-used **SEP-28k** dataset, consisting of *3-second, multi-label audio clips sampled at 16kHz*.

The primary goal is to offer a ready-to-use resource that allows researchers to train and benchmark stuttering detection models on a different acoustic domain without extensive pre-processing. The full processing pipeline is documented and the scripts are included for reproducibility.

### Dataset Statistics

**Total Clips: ~4700**
  - Dysfluent: 2490
  - Fluent: ~2210

**Distribution of Dysfluency Types:**
  - Block: 1425 clips (~30%)
  - Prolongation: 443 clips (~9~%)
  - SoundRep: 1650 clips (~35~%)
  - WordRep: 330 clips (~7~%)
  - Interjection: 930 clips (~19~%)
  - NoStutteredWords: 740 clips (~47~%)

## Instructions

The final processed clips and labels are already provided in the `clips/` directory and the main `labels.csv` file. The following instructions are for users who wish to replicate the data processing pipeline from the raw source files or modify the processing parameters.

### Prerequisites:

- Python 3.x
- Required Python libraries: `numpy`, `pandas`, `librosa`, `soundfile`

You can install the required libraries using `pip`:

```bash
pip install numpy pandas librosa soundfile
```

### Step 1: Process Raw Annotations

The first script, `process_annotations.py`, parses the original UCLASS `.grid` annotation files. It extracts timestamps and identifies five types of dysfluencies (blocks, interjections, prolongations, sound repetitions, word repetitions).

**To run the script:**

Execute the following command from your terminal in the root directory of the dataset:

```bash
python textgrid_to_intprocess_annotationsermediate.py ./annotations/ ./intermediate/
```

This command will read the 18 `.grid` files from the `annotations/` directory and generate 18 corresponding `.txt` files in the `intermediate/` folder. Each `.txt` file contains the timestamps, transcribed text, and detected dysfluency types for one recording.

### Step 2: Slice Audio and Generate Labels

The second script, `audio_slicer.py`, uses the files from the `intermediate/` directory and the original audio from the `audio/` directory to generate the final dataset. It slices the audio into 3-second clips centred on dysfluency events and also extracts an equal number of fluent clips for balance.

**To run the script:**

Execute the following command from your terminal:

```bash
python audio_slicer.py ./intermediate/ ./audio/ ./clips/
```

This command will:

1. Create a subdirectory `clips/clips/` containing all *3,245* sliced `.wav` files.
2. Generate a `labels.csv` file inside the `clips/` directory, containing the file paths and multi-label binary annotations for each clip.
3. Generate a `metadata.json` file inside the `clips/` directory with more detailed information for each clip.

### Directory Structure

The dataset is organised into the following directories:

- `/annotations/` *(18 items)*
    - Contains the original, selected *syllable-level* `.grid` annotation files from the UCLASS dataset.
- `/audio/` *(18 items)*
    - Contains the original `.wav` audio recordings corresponding to the annotation files.
- `/intermediate/` *(18 items)*
    - Holds the pre-processed `.txt` files generated by `textgrid_to_intermediate.py`. This is a transitional format used as input for the final slicing script.
- `/clips/` *(3,245 items + labels)*
    - The final output directory. It contains the `labels.csv`, `metadata.json`, and a `/clips/` sub-folder holding all the sliced 3-second audio clips.
